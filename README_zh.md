# PlotRedox

一款基于 **Rust** 和 **egui** 构建的原生高性能图表数据提取工具。加载图表图片，校准坐标轴，点击数据点，即可将提取的坐标导出为 CSV —— 支持**计算机视觉辅助识别**和内置**脚本 IDE** 进行实时数据分析。

[WebPlotDigitizer](https://github.com/automeris-io/WebPlotDigitizer) 的免费开源替代方案。

## 功能特性

### 核心数字化

- 通过文件对话框、拖放或剪贴板粘贴加载图片
- 4 点坐标轴校准，支持线性和对数刻度
- 多数据系列（分组），支持不同颜色和拖放管理
- 导出提取数据为 CSV
- 撤销 / 重做支持
- 深色和浅色主题
- 多语言界面（English、中文）
- 跨平台（Linux、macOS、Windows）

### 项目管理

- **保存 / 打开项目**为 `.prdx` 文件（ZIP 格式，包含校准数据、数据点、分组和嵌入图片）
- **另存为**创建项目副本
- **新建项目**带有未保存更改保护 —— 丢弃工作前提示保存
- 窗口标题显示项目名称和修改状态
- `Ctrl+S` / `Cmd+S` 快速保存

### 计算机视觉识别

使用基于蒙版的计算机视觉自动检测坐标轴和数据点，无需逐个手动点击。

#### 坐标轴识别

1. 从画布工具栏切换到**坐标轴画笔**模式
2. 使用画笔工具在坐标轴线上绘制蒙版（画笔/橡皮擦，可调画笔大小）
3. 识别引擎检测坐标轴线（方向线拟合）和刻度标记
4. 查看高亮显示的检测结果，然后点击**应用**设置校准点

#### 数据识别

1. 从画布工具栏切换到**数据画笔**模式
2. （可选）在子工具栏中启用**网格去除**，若图表含网格线可提升识别精度
3. 在数据区域上绘制蒙版
4. 引擎进行基于颜色的聚类，分离不同的数据系列
5. 对每个检测到的颜色组，选择**曲线**（连续采样）或**散点**（点检测）模式
6. 智能处理开放曲线与闭合图形（圆、椭圆等），基于弧长闭合检测与自适应组件细化
7. 调整采样点数并预览，然后添加到数据集

#### 网格去除

- 直接集成在**数据画笔**子工具栏中，无需切换独立模式
- 空间中值轮廓检测法去除图表图片中的网格线，提高数据识别精度
- 可调强度滑块（0–1）进行微调
- 实时预览，可切换与原图对比
- 坐标轴检测始终使用原图；网格去除仅影响数据识别

所有识别模式在后台线程运行，保持界面流畅响应。蒙版绘制支持 Shift+点击画直线和约束轴向绘制（水平/垂直锁定）。

### 脚本 IDE

- 内置实时脚本 IDE，基于 [Rhai](https://rhai.rs/)（Rust 嵌入式脚本语言）
- **脚本模板** —— 预置示例可直接使用：
  - 基本语法和函数演示
  - 线性回归
  - 二次（多项式）回归
  - Steinmetz 参数拟合（多元最小二乘）
- **工作区面板** —— 脚本执行后查看所有变量，点击查看数据表
- **导入 / 导出** —— 保存和加载 `.rhai` 脚本文件
- **帮助** —— 内置脚本参考，含语法指南和完整 API 文档（支持中文）

### 数学与分析 API

脚本可使用丰富的内置函数：

| 类别 | 函数 |
|------|------|
| 数学 | `abs`、`sqrt`、`ln`、`log10`、`log2`、`exp`、`pow`、`pow10`、`sin`、`cos`、`tan`、`asin`、`acos`、`atan`、`atan2`、`floor`、`ceil`、`round`、`round_to`、`PI()` |
| 数组 | `sum`、`mean`、`min_val`、`max_val`、`std_dev`、`variance`、`log10_array` |
| 数据 | `col(array, "field")`、`extract_number(string)` |
| 回归 | `linreg(x, y)`、`polyfit(x, y, degree)`、`lstsq(A, b)` |

## 路线图与已知问题

| 状态 | 事项 |
|------|------|
| 🚧 进行中 | **环状/环形曲线识别** — 近期的改进（弧长闭合检测、自适应组件细化）已能较好处理实心圆和椭圆，但**环形（annular）曲线**的识别效果仍不稳定，这是当前正在积极改进的方向。 |

## 从源码构建

从源码构建可确保您运行的是本仓库中的确切代码。这是注重安全的用户推荐的方法。

### 1. 安装 Rust

如果尚未安装 Rust，请访问 [rustup.rs](https://rustup.rs/) 或运行：
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

### 2. 安装系统依赖

**Linux（Debian / Ubuntu）**
```bash
sudo apt-get update
sudo apt-get install -y libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev \
    libxkbcommon-dev libssl-dev libgtk-3-dev
```

**macOS** – 需要 Xcode 命令行工具：`xcode-select --install`

**Windows** – 无需额外依赖。

### 3. 构建并运行
```bash
git clone https://github.com/elechou/PlotRedox.git
cd PlotRedox

# 以优化的 release 模式运行
cargo run --release
```

编译后的二进制文件位于 `target/release/plot-redox`（Windows 上为 `.exe`）。

## 使用方法

### 第 1 步 – 加载图片

打开应用程序并加载图表图片：
- 点击**加载图片**
- 从剪贴板**粘贴**（`Ctrl+V` / `Cmd+V` 或点击"粘贴图片"）
- 将图片文件拖放到窗口

项目中包含示例图片（`sample_plot_1.png`、`sample_plot_2.png`）和示例项目（`sample_plot.prdx`）供测试使用。

### 第 2 步 – 校准坐标轴

**手动校准：**

1. 点击**手动放置坐标轴点**，在图片上点击 **4 个已知参考点**（X 轴两个，Y 轴两个）。
2. 在左侧边栏输入每个参考点的实际值（X₁、X₂、Y₁、Y₂）。
3. 如果坐标轴使用对数刻度，启用**对数 X** / **对数 Y** 复选框。

**自动校准（CV 辅助）：**

1. 在画布工具栏切换到**坐标轴画笔**模式。
2. 使用画笔工具在坐标轴线上绘制蒙版。
3. 引擎自动检测坐标轴和刻度标记。
4. 查看高亮结果，然后点击**应用**设置校准点。

### 第 3 步 – 提取数据点

**手动提取：**

1. 使用画布工具栏切换到**添加数据**模式。
2. 在图表上点击数据点。提取的坐标显示在左侧边栏。

**自动提取（CV 辅助）：**

1. 在画布工具栏切换到**数据画笔**模式。
2. （可选）若图表含网格线，可在子工具栏中启用**网格去除**——调整强度并预览处理效果。
3. 在数据区域上绘制蒙版。
4. 查看检测到的颜色组 —— 选择曲线模式（连续/散点）和点数。
5. 点击**添加为分组**将检测到的点导入数据集。

### 第 4 步 – 整理与导出

- 使用**分组**创建、重命名和设置不同数据系列的颜色。
- 在分组间拖放数据点。
- 点击**导出 CSV** 保存所有数据。

### 第 5 步 – 保存项目

- **Ctrl+S** / **Cmd+S** 保存（或 **文件 → 保存项目**）
- **文件 → 另存为…** 保存到新位置
- 项目保存为 `.prdx` 文件，包含所有数据和嵌入的图片

### 第 6 步 – 使用脚本分析

1. 点击**脚本 IDE** 打开 IDE 面板。
2. 从**脚本模板**选择模板或编写自己的 Rhai 脚本。
3. 点击**运行脚本**执行。结果显示在输出面板；变量显示在工作区面板。
4. 点击**帮助**查看完整 API 参考和语法指南。

## 自定义内置脚本

PlotRedox 提供强大的**脚本自动发现系统**。您不仅限于仓库中提供的预设；可以轻松将自己的代码片段永久添加到 IDE。

### 工作原理

构建系统（`build.rs`）在编译期间自动扫描 `example_scripts/` 目录，并将找到的每个 `.rhai` 文件直接嵌入到应用程序二进制文件中。这意味着您的个人分析脚本将像内置脚本一样出现在**脚本模板**菜单中。

### 添加自定义脚本

1. 进入仓库根目录的 `example_scripts/` 文件夹。
2. 创建新的 `.rhai` 文件（例如 `my_custom_filter.rhai`）。
3. **重新编译**应用程序：`cargo run --release`。

#### 整理技巧

- **菜单顺序**：文件按字母排序。使用数字前缀如 `01_load_data.rhai`、`02_clean_data.rhai` 来控制下拉菜单中的确切顺序。
- **显示名称**：系统自动美化文件名。例如，`03_advanced_log_fit.rhai` 将在 IDE 中显示为 **"Advanced Log Fit"**。

## 项目结构

```
PlotRedox/
├── src/
│   ├── main.rs              # 应用入口点 & eframe 生命周期
│   ├── action.rs            # Action 枚举定义
│   ├── action_handler.rs    # 核心分发逻辑（Action → State 变更）
│   ├── core.rs              # 校准数学和坐标映射
│   ├── state.rs             # 运行时状态 & 可序列化项目数据
│   ├── project.rs           # 项目保存/加载（.prdx ZIP 格式）
│   ├── i18n.rs              # 国际化（英文 / 中文）
│   ├── icons.rs             # Nerd Font 图标常量
│   ├── ui/                  # UI 层（画布、面板、工具栏、对话框）
│   ├── ide/                 # 脚本 IDE（编辑器、工作区、检查器、帮助）
│   ├── script/              # Rhai 脚本引擎 & 数学函数
│   └── recognition/         # CV 识别（坐标轴、数据、蒙版、网格去除）
├── assets/                  # 应用图标 + 字体（更纱黑体 UI SC 用于 CJK）
├── example_scripts/         # 内置脚本模板（.rhai）
├── docs/                    # 脚本参考文档（英文 & 中文）
├── build.rs                 # 脚本嵌入、字体子集化、Windows 图标
├── build_font_subset.rs     # CJK 字体子集化逻辑
├── Cargo.toml               # 依赖和构建配置
└── screenshot.png           # 应用截图
```
